# 🛒 Grocery Store Management System (SQL Project)

## 🚀 Overview
This project demonstrates the design and implementation of a **relational database system** for a grocery store using SQL. It showcases how structured data can be transformed into meaningful business insights through analysis.

The project covers **database design, querying, and analytics**, making it ideal for demonstrating real-world data analyst skills.

![image](https://github.com/kunal1300/Grocery-Store-Management-System-Using-SQL-Project/blob/99c2f8f399f239c041a8e4c44b714e358302e5ac/Images/Er-Diagram.png)
---

## 🎯 Objectives
- Design a **normalized database (3NF)**
- Perform **data analysis using SQL**
- Extract actionable insights on:
  - Customer behavior  
  - Product performance  
  - Sales trends  
- Support **data-driven decision-making**

---

## 🧱 Database Schema
The system consists of **7 interconnected tables**:

- Customers  
- Orders  
- Order Details  
- Products  
- Categories  
- Suppliers  
- Employees  

📌 Key features:
- Foreign key relationships  
- CASCADE constraints  
- Normalized schema (3NF)
